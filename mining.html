Section 02 / Hardware
CPU is king. Here's why.
RandomX was designed to run on your processor, not a GPU farm.
■
CPU mining
RandomX is memory-hard and optimized for general-purpose CPUs. Requires 2GB+ RAM per thread, AES-NI support. AMD Ryzen/EPYC and Intel i7/i9 perform best.
Optimal for Monero
▲
GPU mining
GPUs are structurally disadvantaged with RandomX. Random code execution maps poorly to shader cores. Fewer hashes per watt than CPU. Not recommended since 2019.
Not recommended
◆
RAM matters
Each thread needs 2GB+ of fast memory. RandomX constantly reads/writes a 2GB scratchpad. DDR4-3200+ with tight timings can boost hashrate 10-15%.

Section 04 / Solo vs Pool
Pool mining wins for 99% of miners
Pool mining is recommended for most users — consistent payouts with minimal setup.
Solo mining
You mine alone against the network. At 4,800 H/s vs 5.59 GH/s, expected time to find a block is ~804 days.
+ Full 0.6 XMR reward / No fees / Max privacy
- Extreme variance / May never find a block
Pool mining
Contribute shares to a pool. Steady, predictable payouts proportional to your hashrate. Every few hours to days.
+ Predictable / Low variance / Easy setup
- Pool fee (0-1%) / Trust operator
Best for: Home miners, beginners, anyone under 50 KH/s. This is you.

1. Download XMRig Visit xmrig.com and download the latest release for Windows. Extract the ZIP. Your antivirus will flag it — add an exception. 2. Edit config.json Open config.json in Notepad. Section 07 / Concepts
Hashrate, shares, rewards, payments
How mining pools turn your CPU cycles into XMR.
Hashrate (H/s)
RandomX hashes per second. Higher = more lottery tickets. The network runs ~5.59 GH/s total.
Shares
Proof-of-work fragments submitted to the pool. More shares = bigger slice of the block reward.
PPLNS payments
Pay Per Last N Shares. When pool finds a block, reward splits proportional to recent shares.
