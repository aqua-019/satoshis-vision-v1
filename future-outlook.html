Coming · FCMP++
Full-chain membership proofs
Today, your transaction hides among 16 decoys. With FCMP++, it hides among every transaction ever made — an anonymity set of 100M+ outputs. The largest privacy upgrade in cryptocurrency history.
Ring 16 → Full chain
100M+ anonymity set
Alpha stressnet
kayabanerve lead
CCS-funded 2700+ XMR

How it works: FCMP++ uses Curve Trees — a novel construction combining Selene and Helios curves — to prove your output belongs to the entire set of all outputs, without revealing which one. Proving time is logarithmic: 100M outputs takes only marginally longer than 16. This is the breakthrough making full-chain proofs practical.

What it eliminates: Ring-size heuristic analysis, temporal analysis (guessing real spend by output age), output poisoning attacks (flooding chain with known outputs), and all probabilistic tracing. Every output becomes mathematically equally likely to be the real spend.

Status: Alpha stressnet running. Independent audits by Cypher Stack and additional firms. All code open-source and peer-reviewed. Expected mainnet hard fork Q2-Q3 2026.

Research · Seraphis + Jamtis
Next-generation transaction protocol
Complete redesign of Monero's transaction structure. Modular addresses, improved multisig, forward secrecy, tier-based wallet permissions.
Modular addresses

Carrot bridging
Seraphis replaces CryptoNote-era transaction structure (2013). Modular components upgrade independently without full hard forks. Composable membership proofs. Multisig coordination from n² to n rounds.

Jamtis addressing: Human-readable addresses with forward secrecy. If a view key is compromised in the future, past transactions remain private. Tier-based permissions: find-received, view-balance, view-received, master spend. Merchants can give bookkeepers view-only access without theft risk.

Carrot (Cryptonote Address Rules for Robust Output Tagging): Bridging protocol for backward compatibility. Old wallets keep working during transition. No forced migration.
Development · Cuprate
Rust node implementation
Alternative full node in Rust. Faster sync, memory safety, implementation diversity. A single bug can't take down the entire network.
Rust memory safety
Faster sync
Implementation diversity
Raspberry Pi viable
Why diversity matters: If a critical bug hits C++ monerod, all nodes are vulnerable. Cuprate provides independent implementation — like Ethereum's Geth/Nethermind/Besu diversity. Rust eliminates buffer overflows, use-after-free, data races at compile time.

Practical impact: Faster initial sync → new users operational faster. Lower resource requirements → more full nodes on consumer hardware (Raspberry Pi, old laptops), strengthening decentralization during political pressure.

Counterarguments — eyes open
Liquidity risk: Delistings reduce on/off ramp access. Large positions harder to enter/exit. Spreads increase on DEXs.
Regulatory escalation: Governments could ban possession (not just exchange trading). France briefly considered this. No major economy has yet.
Quantum computing: Theoretical long-term risk to all public-key cryptography. MRL actively researching post-quantum alternatives. Timeline: 10-20+ years.
Protocol risk: Undiscovered cryptographic bugs possible. Mitigated by open-source, multiple audits, and Cuprate diversity.
Bottom line: Counterarguments are real. But demand drivers are substantiated by current data. Ten years of government attempts suggest they can't stop it.
